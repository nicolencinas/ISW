
___________Parrafo 2_________________


1)
Tipo: Funcional.
Prioridad: Esencial.
Descripcion: El cliente registrado podra ingresar una solicitud de pedido de envio.
Detalle: La solicitud de pedido, debera contener la informacion del paquete: origen, destino, peso volumétrico, tipo e informacion adicional.

2)
Tipo: Funcional.
Prioridad: Esencial.
Descripcion: El sistema debera retornar la lista de posibles candidatos para seleccionar. 
Detalle: El sistema retornara una lista con los candidatos disponibles de los cuales el cliente podra seleccionar para realizar el envio.


3)
Tipo: Funcional.
Prioridad: Escencial.
Descripcion: El sistema enviara la notificacion para realizar la entrega.
Detalle: El sistema debera registrar la confirmación de solicitud y enviar la notificación al mensajero con la informacion del pedido.


4)
Tipo: Funcional.
Prioridad: Escencial.
Descripcion: El sistema debe permitir el registro de un nuevo cliente:
Detalle:El cliente podra registrarse al sitio mediante el número de teléfono y empresa.

5)
Tipo: No funcional.
Prioridad: Importante.
Descripcion: El sistema debe ser portable en todos los dispositivos
Detalle: La pagina web del sistema  debe posser una interfaz responsive para poder visualizarse en cualquier dispositivo.

___________Parrafo 4_________________

6)
Tipo: No funcional.
Prioridad: Importante. 
Descripcion: El cliente podra personalizar las parámetros para la busqueda de candidatos

Detalle: El sistama debe permitir la modificacion de parametros de busqueda tales como: vehículo del repartidor, calificacion, distancia respecto al cliente,cantidad de viajes, etc.

___________Parrafo 5_________________


7)
Tipo: No funcional.
Prioridad: Importante.
Descripcion: El sistema debera calcular el tiempo de espera aproximada.
Detalle: El sistema debera utilizar un servicio GPS y en base a las coordenadas obtenidas de los GPS de cada mensajero, debera calcular un tiempo de espera en base a la posicion actual.

8)
Tipo: Funcional.
Prioridad: Importante.
Descripcion: El sistema debe mostrar la ubicacion actual aproximada del mensajero 
Detalle: El sistema debera mostrar la ubicacion actual aproximada en un mapa. Para ello, los mensajeros. contaran con un dispositivo movil que tendra un sensor GPS para consultar la posicion actual de cada uno.


___________Parrafo 6_________________

9)
Requerimiento:RFE
Tipo:Funcional
Prioridad:Esenial
Descripcion:El sistema debe interactuar con un sistema de informacion geografica 
Detalle: para poder ubicar las coordenadas obtenidas del GPS, el sistema 
obtendra un mapa producto de la interaccion con el sistema de informacion 
geografica para poder marcar las posiciones de los mensajeros


___________Parrafo 7_________________

10)
Requerimiento:RFE

Tipo: Funcional

Prioridad: Esencial

Descripcion:El cliente debera poder cancelar una solictud.

Detalle: En el momento que lo requiera el cliente podra cancelar una solicitud 
de pedido  enviandose asi una notificacion al repartidor asignado


11)
Requerimiento: RFE

Tipo:Funcional

Prioridad:Esencial

Descripcion:El mensjero debe poder cancelar una solicitud de pedido.

Detalle: En caso que necesite, el mensajero podra cancelar una solicitud, para ello debera ingresar el motivo. En caso que la solicitud de pedido sea cancelada por el mensajero, el sistema procedera a ofrecer una nueva lista de candidatos mas cercanos para cumplimentar con el pedido



___________Parrafo 8_________________

12)
Requerimiento: RFE

Tipo: Funcional

Prioridad: Esencial

Descripcion: El mensajero debe informar al sistema el inicio y fin de la entrega pactada.

Detalle: Una vez elegido el mensajero, este tiene que informar el inicio del proceso de entrega y posteriormente el fin de la misma y el sistema debera informarlo.


13)
Requerimiento: RFI

Tipo: Funcional

Prioridad: Importante

Descripcion:El sistema enviara un pedido de califiacion del mensajero y del nivel del sevicio en general.

Detalle:Al concluir la entrega, el sistema enviara al cliente un pedido de califiacion del mensajero y servicio en general. Esta informacion sera utilizada como referencia para futuras solicitudes

14)

Requerimiento: RFE

Tipo: Funcional

Prioridad: Esencial

Descripcion:Los repartidores podran informar su disponibilidad

Detalle: los repartidores mediante su dispositivo movil podran informar al 
sistema si estan disponibles o no para ser candidatos para realizar entregas.


___________Parrafo 12_________________
15)

Tipo: Funcional
Prioridad: Importante
Descripción: El operador visualizara las entregas de todos los clientes para poder responder consultas sobre el estado de las mismas.
Detalle: El operador trabajará localmente con el sistema, con esto se asegura que siempre podrá acceder a la información necesaria y responder consultas sobre el estado de las entregas de todos los clientes.

16)
Tipo: Funcional
Prioridad: Importante
Descripción: El operador recibirá notificaciones de perdidas de conexión
Detalle: En el caso de detectarse la pérdida de conexión con el dispositivo móvil de un mensajero asignado a una entrega en curso, el sistema notificará al operador. 

___________Parrafo 13_________________
17)

Tipo: Funcional
Prioridad: Esencial
Descripción: El sistema deberá normalizar las direcciones  para evitar inconvenientes con el destino de entrega.
Detalle: El sistema deberá normalizar las direcciones para evitar inconvenientes con el destino de entrega, para esto deberá interactuar con algún servicio externo que permita la dicha normalización.

___________Parrafo 14_________________

18)

Tipo: Funcional
Prioridad: Deseable
Descripción: Los mensajeros podrán ingresar las incidencias del trafico.
Detalle: Los mensajeros podrán ingresar las incidencias en el tráfico, para permitir al sistema calcular en forma más precisa los tiempos de arribo, a partir de las estadísticas propias. Adicionalmente el sistema interactuara con mas de un servicio  externo para obtener las alertas de tránsito,siempre suponiendo que algunos de ellos estará disponible para brindarnos la información.
