Tipo: Funcional.
Prioridad: Esencial.
Descripcion: El cliente registrado podra ingresar una solicitud de pedido de envio.
Detalle: El cliente debe especificar la informacion del paquete: origen, destino, peso volumétrico, tipo e informacion adicional. Ademas, el sistema debera presentar la lista de posibles candidatos para realizar el envio. 


2)
Tipo: Funcional.
Prioridad: Escencial.
Descripcion: El sistema enviara la notificacion para realizar la entrega.
Detalle: El sistema debera registrar la confirmación de solicitud y enviar la notificación al mensajero con la informacion del pedido.


3)
Tipo: Funcional.
Prioridad: Escencial.
Descripcion: El sistema debe permitir el registro de un nuevo cliente.
Detalle:El cliente podra registrarse al sitio mediante el número de teléfono y empresa.

4)
Tipo: No funcional.
Prioridad: Importante.
Descripcion: La interfaz grafica del sistema tiene que ser adaptable a todos los dispositivos.
Detalle: La pagina web del sistema debe poseer una interfaz responsive para poder visualizarse en cualquier dispositivo (movil o pc).

___________Parrafo 4_________________

5)
Tipo: No funcional.
Prioridad: Importante. 
Descripcion: El cliente podra personalizar las preferencias para la busqueda de candidatos.
Detalle: El sistema debe permitir la eleccion de distintos criterios de busqueda. Estos filtros podran ser: vehículo del repartidor, calificacion, distancia del mensajero respecto al cliente,cantidad de viajes, etc.

___________Parrafo 5_________________


6)
Tipo: No funcional.
Prioridad: Importante.
Descripcion: El sistema debera calcular el tiempo de espera aproximada.
Detalle: El sistema debera utilizar un servicio de posicionamiento. En base a las coordenadas obtenidas de los GPS de cada mensajero, el sistema debera calcular un tiempo de espera en base a la posicion actual.


___________Parrafo 6_________________

7)

Prioridad:Esencial

9)


Tipo:Funcional

Prioridad:Esencial

15)

Tipo: Funcional
Prioridad: Importante

Detalle: El operador trabajará localmente con el sistema, con esto se asegura que siempre podrá acceder a la información necesaria y responder consultas sobre el estado de las entregas de todos los clientes.

___________Parrafo 14_________________

18)

Tipo: Funcional
Prioridad: Deseable
Descripción: Los mensajeros podrán ingresar las incidencias del trafico.
Detalle: Los mensajeros podrán ingresar las incidencias en el tráfico, para permitir al sistema calcular en forma más precisa los tiempos de arribo, a partir de las estadísticas propias. Adicionalmente el sistema interactuara con mas de un servicio  externo para obtener las alertas de tránsito,siempre suponiendo que algunos de ellos estará disponible para brindarnos la información.
